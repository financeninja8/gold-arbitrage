# 套利交易常見問題 FAQ

## 數據源與 API 限制
本系統整合了以下三個黃金價格數據源：

1. **Metals.dev**
   - **限制**：免費版每月 **100 次**請求。
   - **更新頻率**：系統設定為一天更新 3 次 (30天 僅消耗 90 次)，剛好符合限制。
   
2. **MetalPriceAPI**
   - **限制**：免費版每小時 **100 次**請求。
   - **優勢**：頻率額度較高，可作為穩定的備援。

3. **GoldPrice.org**
   - **限制**：公開數據接口，通常無嚴格限制 (約 60次/小時)。
   - **角色**：作為主要的公開即時價格來源。

## 價差計算機制
系統會自動將上述不同來源的報價進行兩兩比對：
- **計算公式**：`利潤 = 賣方出價 (Bid) - 買方要價 (Ask)`
- 若數據源之間的價格差異大於交易成本，系統將會在「套利機會 (Arbitrage Opportunities)」面板顯示該組合。

## 基礎概念

### Q1: 什麼是套利交易？
**A:** 套利交易是利用**同一資產在不同市場的價格差異**來獲取利潤的策略。例如：當黃金期貨在 A 交易所的價格低於 B 交易所時，同時在 A 買入、在 B 賣出，賺取價差。

### Q2: 套利交易的優勢是什麼？
**A:** 
- **風險相對較低**：買賣同時進行，不受單邊漲跌影響
- **收益穩定**：追求確定性的價差利潤
- **系統化執行**：適合程式交易，排除人為情緒

### Q3: 套利交易有風險嗎？
**A:** 是的，主要風險包括：
- **執行風險**：下單延遲可能導致無法同時成交
- **流動性風險**：市場深度不足時難以執行
- **手續費風險**：價差可能被交易成本吃掉
- **滑價風險**：實際成交價格與預期不同

## 套利類型

### Q4: 常見的套利類型有哪些？
**A:** 
1. **跨交易所套利**：同一商品在不同交易所的價差
2. **跨期套利**：同一商品不同月份合約的價差
3. **期現套利**：期貨與現貨之間的價差

### Q5: 什麼是跨期套利？
**A:** 利用同一交易所**不同月份期貨合約**間的價差進行套利。例如：買入近月合約、賣出遠月合約（正價差套利）。

### Q6: 什麼是期現套利？
**A:** 利用**期貨價格與現貨價格**的差異（基差）進行套利。當基差偏離合理範圍時進行交易。

## 實務操作

### Q7: 套利需要多少資金？
**A:** 取決於：
- 交易的商品保證金需求
- 同時持有正反向部位的保證金
- 建議至少準備**預期單筆套利金額的 10-20 倍**作為緩衝

### Q8: 套利交易需要什麼設備？
**A:** 
- 穩定的網路連線
- 低延遲的報價系統
- 程式交易 API（如群益 Capital API）
- 監控多市場價差的軟體

### Q9: 套利機會持續多久？
**A:** 通常只有**數秒到數分鐘**，因為：
- 市場效率會快速消除價差
- 其他套利者也在競爭同一機會
- 需要毫秒級的執行速度

### Q10: 如何計算套利利潤？
**A:** 
```
套利利潤 = (賣出價 - 買入價) × 數量 - 手續費 - 滑價成本
```
需考慮：交易手續費、期交稅、匯率差異（跨市場時）

## 黃金期貨套利

### Q11: 黃金期貨可以在哪些交易所交易？
**A:** 主要交易所：
- **COMEX**（紐約商品交易所）- 國際最大
- **SGE**（上海黃金交易所）
- **TOCOM**（東京商品交易所）
- **台灣期貨交易所** - 黃金期貨

### Q12: 黃金套利需要注意什麼？
**A:** 
- 交易時間差異（不同時區）
- 匯率波動影響
- 各交易所合約規格不同
- 保證金需求較高

### Q13: 如何監測黃金套利機會？
**A:** 
1. 即時連接多個交易所報價
2. 計算價差並設定閾值警報
3. 考慮交易成本後判斷是否有利可圖
4. 使用程式自動化監測

## 風險管理

### Q14: 套利失敗怎麼辦？
**A:** 
- 設定停損機制
- 單邊成交時快速處理另一邊
- 保留足夠的保證金應對浮動虧損
- 分散多種套利策略

### Q15: 如何降低套利風險？
**A:** 
1. 使用高速網路降低延遲
2. 選擇流動性好的商品
3. 設定合理的價差閾值
4. 控制單次交易規模
5. 持續監控市場狀況
